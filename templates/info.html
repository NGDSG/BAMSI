Welcome to BAMSI

What is BAMSI

BAMSI - The BAM Search Infrastructure - is a tool for extracting data from the 1000 Genomes phase 3 set of mapped reads (BAM files). It is a filtering tool - subsets of the ~60TB data can be specied and downloaded via the browser, with no need to download the entire data set.

The filtering is based on SAMtools and offers the same options, as well as additional ones.

How to use this site


Filter jobs are started from the Launcher page.

Files can be selected by population or individual ID, and the filter criteria can be specified.
Once a filter job has been launched, it is given a tracking ID.

*It is very important that you store this ID.* It is the means by which you can track progress and download your results.


The Dashboard page is for monitoring the progress of your job.

Finished tasks can be downloaded from here.




This tool is open for anyone to use. It is however, a protoype, and we can offer no guarantees regarding performance. We recommend using BAMSI for queries of high precision, which result in small sets of data to be downloaded (e.g. specifying a few regions, individuals, or having very strict filter critera). For queries requiring parsing of the entire data set, or yielding large resulting files, performance is subject to resource availability.

Additional information

BAMSI is a cloud-based framework able to leverage compute and data resources from different providers and locations to optimize the efficiency of filtering.

This particular instance is hosted by SNIC Science Cloud , accessing the 1000 Genomes data from a mirror on UPPMAX (Uppsala Multidisciplinary Center for Advanced Computational Science).

BAMSI is also freely available for anyone wishing to set up their own instance of the framework. It is designed to have pluggable components, so compute resources on one or several public or private clouds can be configured to access the data from different mirrors, and the results can be stored in a location of your choice.

Besides improving the access to the 1000 Genomes data set, our intention is also to facilitate and encourage the use of distributed computing platforms for analysis of genetic data. By configuring a distributed storage backend such as HDFS or Swift for the results, post-processing can be performed using resources located close to the data. To this end, we provide the option to download the results in a modified version of the SAM format in which the individual and region information is included in every alignment. The alignments thus become self-contained units, rendering the data suitable for processing with frameworks such as Hadoop or Spark.
